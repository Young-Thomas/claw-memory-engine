# claw-memory-engine 项目规划（两周精简版）
> 核心方向：**CLI高频命令记忆系统**（方向A）
> 开发语言：Python
> 开发周期：2周，可快速落地出Demo

---

## 🎯 项目定位（为什么选这个方向？）
✅ **开发难度最低**：无需对接飞书开放平台，无需复杂的飞书机器人/文档解析开发，两周可完整跑通  
✅ **落地性最强**：完全符合OpenClaw CLI生态，直接可以在终端使用  
✅ **效果最直观**：演示效果好，评测数据容易量化，提效效果肉眼可见  
✅ **项目价值高**：解决开发者真实痛点，属于生产力工具，实用性强

### 核心解决的问题
开发者每天在终端输入大量重复的长命令、复杂参数、项目特定指令，每次都要回忆/翻历史记录，效率极低。我们的系统自动记录用户的命令习惯，提供智能补全和推荐，大幅提升终端操作效率。

## 🔧 核心功能（两周必做）
### 1. 自动记忆
- 静默记录用户所有CLI命令、执行目录、执行状态、时间戳
- 自动统计命令使用频率、常用参数组合
- 按项目目录自动分类记忆，不同项目的命令互不干扰

### 2. 智能推荐
- 输入命令前缀时自动补全完整命令 + 常用参数
- 根据当前目录自动推荐该项目的常用命令
- 支持模糊搜索：用户输入关键词就能找到相关命令，不需要记完整前缀

### 3. 主动管理
- `/remember [别名] [命令]`：主动保存重要命令，支持自定义别名
- `/forget [关键词]`：删除不需要的记忆
- `/list`：查看当前项目的所有记忆命令

### 4. 遗忘机制
- 自动去重相同命令，保留最常用版本
- 长时间未使用的命令自动归档，不占用检索资源
- 新记忆自动覆盖冲突的旧记忆，同时保留历史版本可回溯

## 🚀 最小可运行Demo流程（演示用）
1. 安装后打开终端，cd到项目目录
2. 执行一次复杂的长命令：`kubectl apply -f deployment.yaml -n production --kubeconfig ~/.kube/prod/config`
3. 下次再输入`kubectl ap`，自动补全完整命令，按Tab直接执行
4. 输入`/remember k8s部署` 给这个命令加别名，之后输入`k8s部署`直接执行
5. 切换到另一个项目目录，输入相同前缀会优先匹配当前项目的命令
6. 运行评测脚本，输出提效数据
