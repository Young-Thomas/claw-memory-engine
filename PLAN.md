# claw-memory-engine 项目规划
> 企业级长程协作Memory系统，支持CLI与飞书双端记忆流转

---

## 一、项目定位与核心方向
我们选择 **方向A（CLI高频命令记忆） + 方向B（飞书项目决策记忆）** 双场景切入，覆盖开发者效率+项目管理两大核心企业场景：
- ✅ 跨端统一记忆：CLI终端操作记忆 ↔ 飞书文档/群聊决策记忆 双向流转
- ✅ 上下文感知：记住不同项目、不同场景下的用户习惯与团队决策
- ✅ 主动服务：不需要用户主动查询，在合适场景自动推送相关记忆

## 二、整体架构设计
### 2.1 五层核心架构
```
┌─────────────────────────────────────────────────────────┐
│  记忆应用层 (Application Layer)                          │
│  ├─ CLI端：命令补全、参数推荐、工作流恢复                │
│  ├─ 飞书端：决策推送、上下文提醒、FAQ自动回复            │
│  └─ 管理后台：记忆可视化、权限管理、遗忘规则配置          │
├─────────────────────────────────────────────────────────┤
│  记忆检索层 (Retrieval Layer)                            │
│  ├─ 语义检索：向量相似度匹配 + BM25关键词检索            │
│  ├─ 上下文过滤：按项目/用户/时间/场景维度过滤结果        │
│  └─ 召回排序： freshness（新鲜度）> relevance（相关度）   │
├─────────────────────────────────────────────────────────┤
│  记忆存储层 (Storage Layer)                              │
│  ├─ 向量数据库：存储记忆的语义嵌入（Chroma / Milvus）    │
│  ├─ 关系数据库：存储结构化元数据（SQLite / PostgreSQL）  │
│  └─ 缓存层：热点记忆高频访问缓存（Redis）                │
├─────────────────────────────────────────────────────────┤
│  记忆处理层 (Processing Layer)                           │
│  ├─ 提取：从CLI命令/飞书对话/文档中抽取有效记忆          │
│  ├─ 清洗：去重、过滤无效信息、实体识别                    │
│  ├─ 分类：按记忆类型（命令/决策/偏好/知识）打标签        │
│  └─ 嵌入：生成语义向量用于检索                            │
├─────────────────────────────────────────────────────────┤
│  记忆采集层 (Collection Layer)                           │
│  ├─ CLI端钩子：拦截用户输入命令、执行结果、工作目录       │
│  ├─ 飞书机器人：接收群聊消息、文档更新、事件回调          │
│  └─ 主动录入：用户主动标记/保存的重要信息                │
└─────────────────────────────────────────────────────────┘
```

### 2.2 核心数据流向
```
CLI操作/飞书消息 → 采集层 → 处理层 → 存储层 → 检索层 → 应用层 → 用户
                                 ↓
                          遗忘机制：按访问频率、时间衰减自动清理过时记忆
```

## 四、核心功能详细设计
### 4.1 记忆实体结构
```typescript
interface Memory {
  id: string;                // 唯一ID
  type: 'command' | 'decision' | 'preference' | 'knowledge'; // 记忆类型
  content: string;           // 记忆内容
  metadata: {
    source: 'cli' | 'feishu' | 'manual'; // 来源
    userId?: string;         // 所属用户ID
    project?: string;        // 所属项目
    tags?: string[];         // 标签
    entities?: string[];     // 涉及的实体（人、项目、产品等）
    timestamp: number;       // 创建时间
    lastAccessed: number;    // 最后访问时间
    accessCount: number;     // 访问次数
    expireAt?: number;       // 过期时间（可选）
  };
  embedding: number[];       // 语义向量
}
```

### 4.2 遗忘策略
1. **时间衰减**：记忆权重随时间递减，超过90天未访问自动归档
2. **访问频率**：访问次数越低，权重越低，优先被清理
3. **版本覆盖**：同一主题的新记忆自动替换旧记忆，旧版本保留历史
4. **手动删除**：用户可手动删除/归档不需要的记忆

### 4.3 检索策略
1. **多模态检索**：语义向量匹配 + 关键词匹配 + 元数据过滤
2. **权重排序**：
   - 新鲜度（时间越近权重越高）：40%
   - 相关度（语义匹配度）：35%
   - 访问频率（使用越多权重越高）：25%
3. **上下文过滤**：自动根据当前场景（当前目录、当前讨论话题）过滤无关记忆

## 五、评测方案（满足交付要求）
### 5.1 基础能力测试
| 测试类型 | 测试方法 | 达标要求 |
|---------|---------|---------|
| 抗干扰测试 | 注入1000条无关记忆，1周后查询关键记忆 | 检索准确率≥95%，召回率≥90% |
| 矛盾更新测试 | 先后输入冲突指令：<br>1. "以后周报发给张三"<br>2. "不对，以后周报发给李四" | 最新查询返回"发给李四"，可查询到历史版本 |
| 跨端同步测试 | CLI端保存记忆 → 飞书端查询，飞书端保存记忆 → CLI端查询 | 同步延迟<5s，内容完全一致 |

### 5.2 效能指标测试
| 场景 | 无记忆系统 | 有记忆系统 | 提效目标 |
|------|-----------|-----------|---------|
| 长命令输入 | 平均输入45字符 | 平均输入10字符 | 提效≥75% |
| 历史决策查询 | 平均翻查3个文档/50条聊天记录，耗时2分钟 | 自动推送，耗时<1s | 提效≥99% |
| 项目上下文切换 | 需要重新回忆项目配置/命令，耗时1分钟 | 自动加载对应项目记忆，耗时<1s | 提效≥98% |
